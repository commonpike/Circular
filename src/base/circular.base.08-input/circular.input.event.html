With cc-input elements, the data from the input element is transfered to the target on certain events from the ui, like 'keyup'. You can use the cc-input-event attribute to specify which event is used for a single element: <textarea cc-input cc-input-event="blur:1000" name="data">changes 1 second after blur</textarea> <textarea cc-input cc-input-event="keyup" name="data">changes on keyup</textarea> You can prepend your own definitions (or overwrite the default ones) in Circular.init, like Circular.init({ input : { events : [ { match : 'textarea.slowfeed', event : 'blur:1000' }, { match : 'a.example', event : 'mouseover' } ] } }); The 'match' property is a css selector. The first selector that matches an element is used, so provide the broadest match ('*') last. The 'event' property is a comma separated list of dom events, optionally appended by a timeout (in milliseconds) after a colon.
